Trove Tasks and Volunteers List

This page lists (a) what things currently need doing, and (b) who has volunteered for them.

Volunteers

Eric S. Raymond launched the project. He has general responsibility for the architecture, if only because he's maintaining the design document and project pages. Erik Walthinsen volunteered to do a Postgres DB layer, on 07 Jun 1998.

Kurt Starsinic also volunteered to do a Postgres DB layer, on 15 Jun 1998.

Michael Sobolev volunteered to do database work (backend unspecified), on 16 Jun 1998

M. A. Lemburg volunteered to write the Trove crawler.

Michel Pelletier has volunteered to write the search/browse UI.

Tasks

The shovel:

ESR has done the upper half (TRL parsing, request authentication, and resource retrieval). Lower half (the database and archive interface) still needs to be done.

The crawler:

Not yet started.

The mailbot:

Not yet started. Should be pretty trivial once the shovel is done.

The librarian:

The CGI interface for searching/browsing/downloading is not yet started.

The domain model:

Correct design of a structured-keyword set for use as discriminators is a large nasty problem. Ideally, needs somebody with both intimate knowledge of the domain and library-science/IR skills.

The text search engine

Glimpse? htDIG? Something else? We need somebody with expertise to survey the field and figure out what text search engine best suits our needs.
